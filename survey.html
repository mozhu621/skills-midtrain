Skill Spec Midtraining：文献调研报告

调研日期：2026-06-12 ｜ 调研范围：tool-use/skill SFT、midtraining 方法学、spec/原则学习、skill selection 评测与 OOD 泛化 ｜ 共覆盖 60+ 篇论文/系统

0. 全景概览

这个想法位于三条研究线的交汇处：① tool-use SFT（教模型"怎么调"，已高度成熟但公认只学到行为模仿）、② midtraining（2025 年起被正式定义为预训练与后训练之间的分布桥接阶段）、③ spec/原则显式学习（从 Constitutional AI → Deliberative Alignment → Model Spec Midtraining 一路演进）。三条线各自都很热，但它们的交点——把 skill/tool 的"适用边界、选择理由、失败模式、验证原则"写成规范文本，在 midtraining 阶段注入——目前是空白。

调研最重要的三个结论
  1. "Model Spec Midtraining" 已被 Anthropic 于 2026-05 正式发表（alignment.anthropic.com/2026/msm）：用约 41M token "讨论 spec 的合成文档"做 midtraining，对齐微调数据需求降 40–60×，agentic misalignment 从 54–68% 降到 5–7%，并优于 Deliberative Alignment 基线。但它明确只做了安全/行为规范域，完全没有触碰 tool/skill use。这既是本想法的直接可行性证明，也要求我们把定位写成"MSM 思想在 skill 域的扩展"（增量贡献，而非首创范式）。
  2. 命名无冲突：未发现任何叫 "Skill Spec Midtraining" / "tool spec training" 的工作。但单点要素各有占位者（见 §7 novelty 分析），切割线必须画清楚。
  3. 动机有现成定量证据：LLM 超过 30% 的工具调用是不必要的（SMART）；BFCL irrelevance detection 上 Llama3-8B 仅 ~20%；真实 34k skill 库中 prompted skills 的收益脆弱、接近无 skill 基线（arXiv:2604.04323）。"光把 SKILL.md 放进 context 不够"已经有人替我们证明了。

1. 主题一：Spec / 原则显式学习——从 RLAIF 到 Midtraining

这个主题回答：「知其所以然」式训练是怎么演进到 midtraining 的？

Model Spec Midtraining (MSM) ｜ Anthropic Alignment Science，2026-05
TL;DR：在预训练与对齐微调之间插入一个阶段，训练大量"讨论 Model Spec 内容及其理由"的合成文档，塑造模型后续微调的泛化方向。
核心结果：(a) 相同微调数据 × 不同 spec → 泛化出不同价值观（证明 spec 文本本身在起作用）；(b) agentic misalignment 54–68% → 5–7%，优于 deliberative alignment；(c) 后续 AFT 数据需求降 40–60×；(d) MSM + 无 CoT 的 AFT > 有 CoT 的 AFT，即原则文本可替代部分 CoT 监督。
关系：本想法的母体论文。实验模板（同 SFT 数据 × 不同 spec、数据效率曲线、OOD 评测）可直接复用到 skill 域。
Deliberative Alignment (arXiv:2412.16339) ｜ OpenAI，2024-12
TL;DR：直接把 safety spec 文本教给 o 系列模型，训练其回答前回忆并引用 spec 推理，同时降低 jailbreak 和过度拒答。
关系：证明"教 spec 文本 + 推理时引用" > "从标注样例间接归纳"，且 OOD 泛化更好；MSM 进一步显示 midtraining 注入比这种 SFT 式注入更强。skill 域可设计同款对照（spec 进 SFT vs 进 midtraining）。
Constitutional AI (arXiv:2212.08073) ｜ Anthropic，2022
TL;DR：原则只用来生成偏好信号（RLAIF），模型并不直接学习原则文本本身。
关系：related work 的关键对比点——CAI 学的是原则的"投影"，MSM/本想法学的是原则本身及其 why。
STaR (arXiv:2203.14465，2022) 与 rationale/explanation tuning 系列
TL;DR：训练时带"为什么"的理由能显著改善泛化。
关系：支持 skill spec 数据应包含选择理由与失败模式分析，而非只有规则条文。
Case-Augmented Deliberative Alignment (arXiv:2601.08000) ｜ 2026
TL;DR：spec（成文法）+ 判例式案例联合推理，效果优于纯 spec。
关系：直接启发——skill spec 也应配"判例库"（典型失败案例 + 分析）。

主题综合：原则学习的演进路径是 原则→偏好信号(CAI) → 原则→SFT推理链(DA) → 原则→midtraining语料(MSM)，每一步都让模型对原则的掌握更"深"（更靠近表征层）、数据效率更高。本想法是把这条已验证的演进路径平移到 skill 域。

2. 主题二：Midtraining 方法学——为什么是这个阶段

这个主题回答：为什么 spec 应该放 midtraining，而不是 SFT 或 prompt？

Midtraining Bridges Pretraining and Posttraining Distributions (arXiv:2510.14865) ｜ CMU，2025
TL;DR：midtraining 的机制是"分布桥接"——为 posttraining 提供更近的初始化；收益在离预训练分布最远的领域（代码、数学）最大，并且优于单纯 continued pretraining。
关系：tool 调用格式/skill 使用决策恰是离预训练分布很远的领域 → 理论上预测 skill-spec midtraining 收益显著。
A Survey on LLM Mid-Training (arXiv:2510.23081，美团/北大) ＋ Mid-Training: A Survey (arXiv:2510.06826) ｜ 2025
TL;DR：正式定义 mid-training：中等量级数据/算力，在保持基础能力同时系统性增强指定能力。 术语与分类锚点。
OLMo 2 (arXiv:2501.00656，Ai2) ＋ MiniCPM (arXiv:2404.06395，清华/面壁)
TL;DR：开源界实证——退火期注入指令/高质量数据显著拉升下游；MiniCPM 显示 decay 期混 SFT 数据优于"训完再 SFT"。
关系："两阶段注入 > 单阶段 SFT"的最直接量化证据；也给出实操配方（在学习率退火段注入）。
Physics of Language Models 3.1 (arXiv:2309.14316，Allen-Zhu & Li)
TL;DR：知识必须在预训练期经过充分增强（改写、多样表述）才能被抽取，否则后续指令微调救不回来（0% 抽取率）。
关系：skill 的适用边界属于"知识"→ 必须在 midtraining 注入并做多样化改写，不能指望 SFT。直接指导数据合成策略。
Does Fine-Tuning on New Knowledge Encourage Hallucinations? (arXiv:2405.05904，Google/Technion，2024)
TL;DR：SFT 期注入新知识学得慢，且学会之后幻觉率上升。
关系：直接 SFT 教 skill 边界可能诱发"幻觉式调用"（编造工具适用性）——为"midtraining 而非 SFT"提供风险侧论据。
Pre-Instruction-Tuning (PIT) (arXiv:2402.12847，FAIR) ＋ Instruction Pre-Training (arXiv:2406.14491，微软)
TL;DR：调整知识与指令数据的注入顺序（先 QA 后文档）能让知识吸收提升 17.8%——存在显著的顺序/课程效应。
关系：支持"spec 先于轨迹"的课程设计假设（RQ3 的机制基础）。

主题综合：四条独立证据链（分布桥接机制、退火期注入实证、知识抽取的预训练依赖、SFT 注入新知的幻觉风险）一致指向同一结论："skill 为什么/何时用"这类陈述性知识应在 midtraining 注入，SFT 只负责把它接到行为上。这是 proposal 立论的方法学支柱。

3. 主题三：Tool-use SFT 主流配方——它们到底教了什么

这个主题回答：现有 tool SFT 的天花板在哪、为什么"只教 how"？

工作arXiv / 年份一句话教了什么 / 缺什么
Toolformer2302.04761 / 2023, Meta自监督插 API 调用，按"是否降 LM loss"过滤"何时调用"靠 perplexity 启发式，无显式理由
Gorilla2305.15334 / 2023, BerkeleyGPT 合成 API 指令数据做 SFT对文档浅层匹配，易幻觉不存在的 API
ToolLLM / ToolBench2307.16789 / 2023, 清华16k RapidAPI + DFS 决策树轨迹蒸馏模仿轨迹表面形式；其 G1→G3 OOD 阶梯反而成了标准评测
APIGen / xLAM2406.18518, 2409.03215 / 2024, Salesforce可执行验证 + 多级过滤的函数调用数据流水线教格式正确性与参数填充，不教选择理由
ToolACE2409.00920 / 2024, 华为 (ICLR'25)多智能体自演化合成 26k API、50 万对话，双层验证规模与多样性的 SOTA 配方，仍是行为克隆
Hammer2410.04587 / 2024, OPPO/上交function masking + 无关函数增强已触及"对无关工具敏感"，但只在数据扰动层面，无显式 spec
ToolGen2410.03439 / 2024工具变 vocab token，文档"记忆"进参数证明"工具知识入权重"可行且优于 in-context 文档管线
Granite-FC2407.00121 / 2024, IBM函数调用拆成粒度子任务多任务学习"分解工具能力"先例，但仍是行为层不是理由层

主题综合：该赛道的公认共识（多篇综述 + Nemotron-Tool-N1 等）：SFT 记住轨迹而非内化推理——对新工具/扰动描述脆弱、工具幻觉、过度调用。并且有量化证据：只在近测试分布训练时 OOD 性能可跌 67%。这正是 proposal 的"现状批判"段落，且 xLAM/ToolACE 配方应作为我们 pipeline 中的标准 SFT 阶段（保持该阶段不变，只动 midtraining，才能干净归因）。

4. 主题四：教"边界 / 何时不用 / 验证"的已有尝试（novelty 威胁区）

这个主题回答：谁已经在教 spec 的某个单点要素？我们和他们的切割线在哪？

SMART / SMARTAgent (arXiv:2502.11435) ｜ UIUC，ACL'25 Findings ｜ 最强 novelty 威胁
TL;DR：量化 LLM >30% 的调用是不必要的；用 SMART-ER 数据 SFT 让推理链在参数知识与工具间显式切换，调用量降 24%、性能升 37%。
切割线：只覆盖"用 vs 不用"单一维度；是任务级 SFT，不是 midtraining；rationale 是逐例 CoT 而非稳定的规范文本。它同时是我们最好的动机证据 + 必比 baseline。
Agent-FLAN (arXiv:2403.12881) ｜ 上海AI Lab，ACL'24 Findings
TL;DR：发现 agent 训练把格式与推理纠缠并引入幻觉；显式构造负样本教模型"何时不该调用"，并建 Agent-H 幻觉基准。
切割线：用负样本隐式教边界，没有 spec 式的"为什么"。必引 + 强 baseline，Agent-H 可直接用于评测。
GenTool (arXiv:2502.18990) ｜ ACL'25 Findings
TL;DR：专攻 unseen tool 泛化（Zero-to-One / Weak-to-Strong 两维），靠理解相似工具功能差异的两阶段微调取得泛化。
切割线：与我们"相似 skill 区分"要素最接近的训练工作，但仍是 SFT 内的监督设计、无规范文本。其 OOD 协议值得复现（RQ4）。
When2Call (arXiv:2504.18851) ｜ NVIDIA，NAACL'25
TL;DR：测"调用 / 追问澄清 / 承认不能"三分决策，SOTA 模型仍有大缺口；偏好优化优于普通 SFT。 RQ1 评测 + 其训练集可作对照。
Relign (arXiv:2412.04141)：扩展动作空间加入"犹豫/澄清"，对齐式降工具幻觉。 ＋ The Reasoning Trap (arXiv:2510.22977)：增强推理反而放大工具幻觉，现有缓解手段牺牲能力。
后者是"需要更根本的训练阶段解法"的绝佳论据。
文档侧改进（推理时）：Tool Docs Enable Zero-Shot Tool-Usage (2308.00675，"只给文档 ≈ few-shot demo，工具多时文档更优")；EasyTool (2401.06201，重写文档即提性能)；DRAFT (2410.08197)。
共同构成必须打败的 "spec 放 context" baseline；同时佐证 spec 内容本身有信息增益。ToolTweak (2510.02554) 显示模型对描述措辞脆弱 → "spec 入权重"可主张鲁棒性优势。
调用后验证：CRITIC (prompt 级自我修正)；CRITICTOOL (错误反思评测)。
"验证"作为能力存在研究，但没人把验证原则写成 spec 来学——这是我们五要素里最空白的一个。

5. 主题五：Agent Skills 生态与 skill 内化（2025–2026 热点）

这个主题回答：skill 作为一等公民的生态长什么样？内化赛道卷到哪了？

Anthropic Agent Skills / SKILL.md ｜ 2025-10 发布、2025-12 开放标准
TL;DR：skill = YAML 元数据 + 指令 + 脚本的文件夹，渐进式加载；已被 Codex CLI、Gemini CLI 等采纳。
关系：这是推理时注入 skill spec；我们做的是把它训练进权重——天然对照组，而且 SKILL.md 格式可直接作为我们 spec 的种子格式。
How Well Do Agentic Skills Work in the Wild? (arXiv:2604.04323) ｜ 2026
TL;DR：在 34k 真实 skill 库上，检索使用 skills 的收益脆弱、常接近无 skill 基线。
关系：现成的 motivation 证据——光把 SKILL.md 放进 context，模型不会用；需要训练侧干预。
Agentic CPT / AgentFounder (arXiv:2509.13310) ｜ 阿里通义，2025
TL;DR：首次系统提出 agentic 中间训练阶段，证明 mid-training 注入 agentic prior 能缓解 post-training 的优化张力。
关系："在 SFT 前加 agent 向训练阶段"的直接可行性证据；但其数据是行为轨迹，不是 skill 规范——和我们正交，也是 baseline (e)。同路线还有 daVinci-Dev (2601.18418，软件工程域)。
Skill 内化赛道（2026 上半年密集出现）：SKILL0 (2604.02268)、SkillC (2605.27899)、Skill-SD (2604.10674)、Trace2Skill、SIRI——"训练时用 skill、推理时零依赖"。
关系 + 风险：都从轨迹/RL 信号内化行为，无人从规范文本学习理由。赛道升温快（2026.4–5 三连发），建议尽快占坑，related work 以"轨迹内化 vs 规范理解"为轴切割。综述锚点：SoK: Agentic Skills (2602.20867)、Agent Skills Survey (2602.12430)。

早期源头：Voyager (2305.16291，skill library 开山，in-context)；行为蒸馏系：AgentTuning (2310.12823)、FireAct (2310.05915)、AgentInstruct (2407.03502)。

7. 横向综合：共识、张力、空白与 novelty 分析

7.1 共识

  • 行为克隆式 tool SFT 学到表面形式，OOD 脆弱、幻觉、过度调用——三个赛道（数据合成、对齐、评测）的论文都承认。
  • "何时不用"是当前最大的能力缺口（MetaTool、When2Call、BFCL irrelevance、SMART 多方印证）。
  • 原则/spec 显式学习 > 从样例归纳，且注入越早（越靠 midtraining）效果与数据效率越好（DA → MSM 的演进；MiniCPM/OLMo2/PIT 旁证）。

7.2 张力

  • 入权重 vs 入上下文：ToolGen 证明工具知识入权重可行且更优；但 docs-in-context 路线（2308.00675、EasyTool、Agent Skills 生态）工程上更灵活。我们的实验必须正面回答这个张力（baseline d）。
  • SFT 提升 in-distribution 但伤 OOD（GenTool 等）vs "训练才能根治"——支持把"可迁移的 spec 推理模式"而非"具体工具记忆"作为训练目标。
  • 增强推理放大工具幻觉（Reasoning Trap）vs rationale 改善泛化（STaR）——提示 rationale 的来源很重要：逐例生成的 CoT ≠ 稳定的规范文本。

7.3 空白（= 我们的空间）

  • 无人把「适用边界 + 何时不用 + 相似 skill 区分 + 失败模式 + 调用后验证」五要素写成规范文本并以 midtraining 注入。
  • "调用后验证原则"作为训练内容完全空白（只有 prompt 级 CRITIC 和评测级 CRITICTOOL）。
  • tool SFT 的数据效率 scaling 曲线研究很少——RQ3 本身就能成为一个贡献点。

7.5 风险提示

  • 时间窗口：skill 内化赛道 2026.4–5 三连发，MSM 发布后大概率有人想到同样的扩展。建议 3 个月内出 workshop/preprint 占坑。
  • 故事崩塌点：若 spec midtraining 打不过"spec 全文放 context"（等 token 预算下），核心主张不成立——这是必须最先跑的实验。
